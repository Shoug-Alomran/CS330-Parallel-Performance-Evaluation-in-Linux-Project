Performance Analysis

Interpreting Parallel Execution Behavior

Key Observations

Critical Finding

Process-based parallelism using fork() demonstrated performance degradation rather than improvement for the tested configurations. This counterintuitive result highlights the significant overhead associated with process creation and management for moderate-sized computational problems.

Overhead Sources

  • Process Creation: fork() system call invocation and address space duplication
  • Context Switching: OS scheduler overhead when switching between processes
  • Memory Duplication: Copy-on-write mechanisms for process memory
  • Synchronization: wait() system calls for process coordination
  • Cache Invalidation: Loss of cache locality when distributing work

System Limitations

  • CPU Core Count: 4-core limit creates contention beyond 4 processes
  • Virtualization Layer: Additional abstraction layer adds overhead
  • ARM Architecture: aarch64 may have different performance characteristics
  • Memory Bandwidth: Shared memory access creates contention

Task vs. Data Parallelism Analysis

Characteristic Task Parallelism (fork() processes) Data Parallelism (Thread-based) Recommendation
Creation Overhead High (Full process duplication) Low (Shared address space) Use threads for fine-grained parallelism
Memory Usage High (Separate address spaces) Low (Shared memory) Processes better for isolation, threads for efficiency
Synchronization Complex (IPC mechanisms) Simpler (Shared variables) Threads easier for shared data access
Fault Isolation Excellent (Process boundaries) Poor (One fault affects all) Processes better for reliability
Scalability Limited (High overhead) Better (Lower overhead) Hybrid approach for large-scale systems

Amdahl's Law Application

According to Amdahl's Law, the maximum speedup achievable through parallelization is limited by the sequential portion of the program:

Speedup_max = 1 / (f + (1-f)/P) where f is the sequential fraction

For Our Program

If we estimate f (sequential fraction including overhead) at 0.7:

With P=4: Speedup_max = 1 / (0.7 + 0.3/4) = 1 / 0.775 = 1.29×

Actual observed: 0.39× (much lower due to additional overheads)

Theoretical vs. Actual

The significant discrepancy between theoretical maximum and actual speedup indicates substantial parallelization overhead not accounted for in simple models, including:

  • Process creation/destruction time
  • Memory allocation overhead
  • Synchronization delays
  • Cache coherence penalties

Conclusions & Recommendations

Synthesizing Experimental Findings

Executive Summary

Process-based parallelism using fork() demonstrates significant overhead penalties for moderate-sized computational problems. While theoretically capable of improving performance, practical implementation reveals that the costs of process creation, memory duplication, and synchronization often outweigh parallelization benefits for problems below a certain complexity threshold.

Key Conclusions

Performance Findings

  • Process creation overhead dominates for small-to-medium problem sizes
  • Optimal process count depends on problem size and system resources
  • Performance scaling is sub-linear and can be negative
  • Overhead constitutes 200-300% of serial execution time

System Insights

  • Virtualization adds measurable but consistent overhead
  • ARM architecture performs comparably for this workload
  • Memory bandwidth becomes bottleneck with multiple processes
  • Cache locality loss significantly impacts performance

Complete Source Code

matrix_fork.c - Parallel Matrix Multiplication
Lines: 56 | Size: 2.1 KB 
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>
#include <time.h>

// ============================================================================
// CONFIGURATION PARAMETERS
// ============================================================================
#define N 600          // Matrix dimension (tested: 600, 800, 1200, 1800, 2400)
#define PROCS 4        // Number of child processes (tested: 1, 2, 4)

// ============================================================================
// GLOBAL MATRIX DECLARATIONS
// ============================================================================
double A[N][N], B[N][N], C[N][N];

// ============================================================================
// FUNCTION: initialize_matrices()
// PURPOSE:  Populates matrices A and B with random values, zeroes matrix C
// ============================================================================
void initialize_matrices() {
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            A[i][j] = rand() % 10;      // Random values 0-9
            B[i][j] = rand() % 10;      // Random values 0-9
            C[i][j] = 0;                // Initialize result matrix
        }
    }
}

// ============================================================================
// FUNCTION: multiply(start, end)
// PURPOSE:  Computes matrix multiplication for rows [start, end)
// ============================================================================
void multiply(int start, int end) {
    for (int i = start; i < end; i++) {
        for (int j = 0; j < N; j++) {
            double sum = 0;
            for (int k = 0; k < N; k++) {
                sum += A[i][k] * B[k][j];
            }
            C[i][j] = sum;
        }
    }
}

// ============================================================================
// FUNCTION: main()
// PURPOSE:  Coordinates parallel execution and measures performance
// ============================================================================
int main() {
    // Seed random number generator
    srand(time(NULL));
    
    // Initialize matrices with random values
    initialize_matrices();
    
    // Start timing
    clock_t start_time = clock();
    
    // Calculate rows per process (with handling for remainder)
    int rows_per_proc = N / PROCS;
    
    // Create child processes
    for (int p = 0; p < PROCS; p++) {
        pid_t pid = fork();
        
        if (pid == 0) {  // Child process
            // Calculate this process's row range
            int start = p * rows_per_proc;
            int end = (p == PROCS - 1) ? N : start + rows_per_proc;
            
            // Perform assigned computation
            multiply(start, end);
            
            // Child process exits after completing its work
            exit(0);
        }
        // Parent continues to create more processes
    }
    
    // Parent waits for all child processes to complete
    for (int p = 0; p < PROCS; p++) {
        wait(NULL);
    }
    
    // Stop timing and calculate elapsed time
    clock_t end_time = clock();
    double time_taken = (double)(end_time - start_time) / CLOCKS_PER_SEC;
    
    // Output execution time
    printf("Execution Time: %.3f seconds\n", time_taken);
    
    return 0;
}
