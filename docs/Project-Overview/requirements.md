# Project Requirements

This section defines the technical and academic requirements for the CS330 course project.

These requirements apply across all project phases unless explicitly stated otherwise.

</div>
</div>

---

## Environment Requirements

<div class="grid cards" markdown>

-   :material-linux: **Linux Environment**

    ---
    The project must be completed in a Linux environment (Ubuntu recommended) to ensure consistency and reproducibility.

-   :material-monitor-dashboard: **Virtual Machine**

    ---
    A virtual machine platform (e.g., VirtualBox) must be used to standardize the execution environment.

-   :material-memory: **System Resources**

    ---
    The system must provide sufficient CPU and memory resources to execute experiments reliably.

</div>

---

## Programming Requirements

<div class="grid cards" markdown>

-   :material-language-c: **C Programming Language**

    ---
    All implementation and experimentation must use the C programming language.

-   :material-tools: **GCC Compiler**

    ---
    Programs must be compiled using `gcc`, following correct compilation practices.

-   :material-source-fork: **System-Level Concepts**

    ---
    Proper use of process creation (`fork()`), synchronization (`wait()`), and structured memory handling is required.

-   :material-code-braces: **Base Source Code**

    ---
    The instructor-provided source code serves as the baseline for experimentation.

</div>

---

## Execution & Experimentation

<div class="grid cards" markdown>

-   :material-play-circle-outline: **Compilation & Execution**

    ---
    The program must be compiled and executed directly in the Linux environment.

-   :material-chart-line: **Performance Measurement**

    ---
    Execution time must be measured across multiple configurations (e.g., varying `N` and `PROCS`).

-   :material-tune: **Controlled Modifications**

    ---
    Changes are limited to configurable parameters such as matrix size and number of processes.

-   :material-repeat: **Repeated Trials**

    ---
    Multiple runs must be performed to ensure stable and reliable timing results.

</div>

---

## Documentation Requirements

<div class="grid cards" markdown>

-   :material-file-document-outline: **Environment Description**

    ---
    Clear explanation of the virtual machine and Linux setup.

-   :material-flask-outline: **Experimental Methodology**

    ---
    Description of testing strategy, parameter selection, and measurement procedure.

-   :material-table: **Results Presentation**

    ---
    Performance data must be presented using structured tables and/or graphs.

-   :material-chart-bar: **Analytical Discussion**

    ---
    Written analysis explaining observed performance trends and scalability behavior.

-   :material-format-align-left: **Professional Formatting**

    ---
    Reports must be well-structured, clear, and suitable for academic evaluation.

</div>
